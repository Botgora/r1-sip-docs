# 挂断和清理

呼叫结束路径很多，但清理动作应该统一。不要只依赖某一个事件完成清理。

## 座席主动挂断

座席点击挂断时：

1. Web 端先离开 RTC 频道。
2. 向 `d/{appid}/{device_id}/stop` 发布 STOP 指令。
3. 后端更新通话记录。
4. 后端释放设备占用。
5. 前端恢复到可发起下一通电话的状态。

STOP 最小消息：

```json
{
  "appid": "your_app_id"
}
```

## 设备或手机侧挂断

设备上报 `HANGUP` 时：

1. Web 端离开 RTC。
2. 读取 `cause`、`duration_sec`、`billsec`。
3. 更新通话记录。
4. 释放设备占用。
5. UI 展示结束原因。

## 呼叫失败或超时

收到 `ERROR` 或等待状态超时时：

1. 停止等待状态。
2. 必要时发布 STOP。
3. 离开 RTC。
4. 释放设备占用。
5. 展示明确失败原因。

建议接入方对 CALL 后等待状态设置业务超时。demo 中呼叫等待超时为 60 秒，具体值可按业务调整。

## 页面关闭和异常退出

浏览器关闭、刷新、网络断开时，前端不一定能可靠完成清理。因此设备占用必须有后端超时兜底。推荐保留服务端定时任务或会话心跳，把过期占用自动释放。
