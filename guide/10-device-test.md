# 真设备联调

Web 集成完成后，需要拿一台真实智能设备验证完整链路。联调时不要只看 Web 日志，也要结合设备、配网 APP、设备管理平台和主叫手机状态一起排查。

## 准备工作

### 1. 获取设备

先准备一台可用于联调的智能设备，并确认设备已经完成出厂必要配置。至少需要能拿到：

- 设备实物。
- 设备 SN 或序列号。
- Device ID。格式通常为 `{product_id}-{serial_num}`，例如 `ACP-SP2617XXXXX1`。
- 设备供电方式和必要线材。

如果设备无法被配网 APP 扫描，或者无法出现在设备管理平台中，需要先确认设备是否已完成出厂写入和平台侧预创建。

### 2. 获取配网 APP

联系设备提供方获取配网 APP 安装包。配网 APP 用于完成设备添加、网络配置和用户设备绑定。

注意：设备当前支持 Wi-Fi 和网口联网。即使使用网口，仍然需要通过配网 APP 走设备添加流程，因为 APP 的作用不只是写入 Wi-Fi 信息，也是把设备绑定到当前用户账号下的入口。

### 3. 准备账号

准备一个可登录配网 APP 的终端用户账号。这个账号通常由经销商或管理员在设备管理平台中创建。

联调前确认：

- 账号已创建。
- 账号处于启用状态。
- 已拿到手机号、初始密码等登录信息。
- 后续 Web 座席系统查询设备列表时，使用的是这个账号对应的手机号。

## 联调路径

1. 阅读 [设备介绍](./10-device-overview.md)，确认设备外观、指示灯和接口。
2. 安装并登录配网 APP。
3. 让设备进入配网或绑定流程。
4. 按 [APP 配网介绍](./10-app-provisioning.md) 完成设备绑定和网络配置。
5. 如需升级固件，按 [OTA 升级](./10-ota-upgrade.md) 完成版本升级。
6. 在设备管理平台确认设备出现在当前账号下。
7. 确认设备在线、主叫号码可用。
8. 准备一部已插 SIM 卡的手机，并让手机蓝牙连接设备。
9. 回到 Web 座席系统，选择设备并发起测试呼叫。

## 呼叫前检查

- 设备已绑定到当前账号。
- 设备在设备管理平台可查询。
- 设备状态满足业务可用要求。
- 主叫号码已上报并审核通过。
- 手机蓝牙已连接设备。
- 手机已插 SIM 卡。
- Web 座席系统能成功占用设备。
