# 理解产品如何工作

智能设备人工呼出不是一个单纯的 Web SDK 集成。它由设备、设备管理平台、接入方 Web 座席系统、MQTT 信令和声网 RTC 音频共同完成。

## 各组件的职责

| 组件 | 职责 |
| --- | --- |
| Web 座席端 | 让座席选择设备、输入被叫号码、发起/挂断通话、展示状态，并接入 RTC 音频 |
| 接入方后端 | 查询设备、占用设备、获取 MQTT Token、生成 RTC Token、维护业务通话记录 |
| 设备管理平台 | 管理设备归属、设备状态、主叫号码、审核状态和账号下设备列表 |
| 智能设备 | 接收呼叫指令，通过已连接手机拨号，并把手机侧通话音频桥接到 RTC |
| MQTT | 承载呼叫指令、挂断指令和设备通话状态 |
| 声网 RTC | 承载 Web 座席与智能设备之间的实时语音 |
| 配网 APP | 完成设备添加、网络配置和用户设备绑定 |

## 为什么 Web 座席要先选择设备

一台智能设备对应一个实际拨号能力。Web 座席发起呼叫时，必须知道哪台设备负责拨号，并确认这台设备属于当前账号、已启用、主叫号码可用、当前没有被其他座席占用。

因此，呼叫的第一步不是连 MQTT，也不是加入 RTC，而是从设备管理平台查询当前账号下的候选设备。

## 为什么必须有接入方后端

生产集成中，Web 前端不应该直接持有服务端密钥，也不应该只用前端变量判断设备是否被占用。接入方后端需要承担这些职责：

- 使用平台分配的 `app_id` 和 `app_secret` 换取设备管理平台 `access_token`。
- 查询账号设备列表，并按业务规则过滤可用设备。
- 在座席点击呼叫时占用设备，避免并发冲突。
- 调用 MQTT Token 接口，向前端返回短期可用的 MQTT 连接凭证。
- 使用声网 App ID 和 App Certificate 生成声网 RTC Token。
- 在呼叫结束、失败、超时或异常退出时释放设备占用。

## 配网和绑定的关系

设备当前支持 Wi-Fi 和网口联网。即使设备使用网口联网，也仍然需要通过配网 APP 走设备添加流程，因为配网 APP 不只是写入 Wi-Fi 信息，也是把设备绑定到当前用户账号下的入口。

如果跳过 APP，设备可能已经联网，但不会出现在用户设备列表里，Web 座席系统也无法按账号查询和选择这台设备。
